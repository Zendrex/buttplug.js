## Protocol Types

### OutputType

Supported output actuator types in the Buttplug v4 protocol.

```ts
type OutputType =
  | "Vibrate"
  | "Rotate"
  | "RotateWithDirection"
  | "Oscillate"
  | "Constrict"
  | "Spray"
  | "Temperature"
  | "Led"
  | "Position"
  | "HwPositionWithDuration";
```

### InputType

Supported input sensor types in the Buttplug v4 protocol.

```ts
type InputType = "Battery" | "RSSI" | "Pressure" | "Button" | "Position";
```

### InputCommandType

Sensor command operation sent to the server.

```ts
type InputCommandType = "Read" | "Subscribe" | "Unsubscribe";
```

### OutputCommand

Tagged union of output command payloads, discriminated by actuator type key. Each variant is a single-key object mapping the output type to its data shape.

Variants: `Vibrate`, `Rotate`, `RotateWithDirection`, `Oscillate`, `Constrict`, `Spray`, `Temperature`, `Led`, `Position`, `HwPositionWithDuration`.

### ClientMessage

Tagged union of all client-to-server messages. Each variant is a single-key object wrapping a specific message schema.

Variants: `RequestServerInfo`, `StartScanning`, `StopScanning`, `RequestDeviceList`, `Ping`, `Disconnect`, `StopCmd`, `OutputCmd`, `InputCmd`.

### ServerMessage

Tagged union of all server-to-client messages.

Variants: `ServerInfo`, `Ok`, `Error`, `DeviceList`, `ScanningFinished`, `InputReading`.

### InputReading

Server message delivering a sensor reading from a device. Contains `DeviceIndex`, `FeatureIndex`, and a `Reading` union discriminated by sensor type.

### ServerInfo

Handshake response from the server. See [ButtplugClient properties](/docs/client/api/client#properties).

### InputData

Tagged sensor data union discriminated by sensor type key. Each variant maps a sensor type to `{ Value: number }`.

Variants: `Battery`, `RSSI`, `Pressure`, `Button`, `Position`.

## Pattern Types

Documented on the [PatternEngine](/docs/patterns) page:

| Type | Description |
|------|-------------|
| `PresetPattern` | Preset pattern descriptor |
| `CustomPattern` | Custom keyframe pattern descriptor |
| `PatternDescriptor` | Union of `PresetPattern \| CustomPattern` |
| `Keyframe` | Animation keyframe with value, duration, easing |
| `Track` | Sequence of keyframes bound to a device feature |
| `PatternPlayOptions` | Playback options (intensity, speed, loop, callbacks) |
| `PatternInfo` | Read-only snapshot of an active pattern |
| `PresetInfo` | Metadata for a built-in preset |
| `PresetName` | `"pulse" \| "wave" \| "ramp_up" \| "ramp_down" \| "heartbeat" \| "surge" \| "stroke"` |
| `Easing` | `"linear" \| "easeIn" \| "easeOut" \| "easeInOut" \| "step"` |
| `StopReason` | `"manual" \| "complete" \| "timeout" \| "error" \| "disconnect" \| "deviceRemoved"` |
